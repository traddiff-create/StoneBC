# StoneBC — Codex Instructions

## Project Overview

Stone Bicycle Coalition iOS app — community bike co-op in Rapid City, SD.
Config-driven so other bike co-ops can fork it. Local-first data, no user auth.

**Bundle ID:** `com.traddiff.StoneBC` | **Target:** iOS 17.0 | **Version:** 0.8

## Tech Stack

- Swift 5, SwiftUI, MapKit, CoreLocation, CoreMotion, WeatherKit, HealthKit
- ActivityKit, AVFoundation, MultipeerConnectivity, SQLite3, ImageIO, PhotosUI
- Architecture: MVVM with `@Observable` AppState (single source of truth)
- Data: Bundled JSON (local-first) + optional WordPress REST API sync
- Config: `config.json` — feature flags, branding, contact, data URLs

## App Structure

5-tab layout: **Home → Routes → Bikes → Radio → More**

- `AppState.swift` — all data loads here (`@Observable`)
- `BCDesignSystem.swift` — all reusable UI components (FilterChip, badges, PressableButtonStyle)
- `AppConfig.swift` — config-driven settings from config.json
- `Radio/` — all Rally Radio files (MultipeerConnectivity push-to-talk)

**Never hardcode** coalition name, email, or colors — always read from `config.json`.

## Key Files

| File | Purpose |
|------|---------|
| `StoneBCApp.swift` | Entry point |
| `ContentView.swift` | Root → TabContainerView + AppState injection |
| `TabContainerView.swift` | 5-tab navigation |
| `AppState.swift` | Central `@Observable` state — data + loading logic |
| `BCDesignSystem.swift` | Design tokens: colors, typography, reusable components |
| `config.json` | Feature flags, branding, contact info, data URLs |
| `bikes.json` | Bike inventory (source: `inventory/bikes.json`) |
| `routes.json` | 42 GPX routes (531KB, generated by `Scripts/process_routes.py`) |

## UI Patterns

- Section headers: 10pt semibold, tracking 1, secondary color
- Cards: `BCSpacing.md` padding, `cardBackground`, 12pt cornerRadius
- Lists: `LazyVStack` with `BCSpacing.md` horizontal padding
- Use `BCDesignSystem` components — don't reinvent FilterChip or badges

## Data Management

Content is owner-managed (no user-generated content):
- Bikes: edit `inventory/bikes.json` → copy to `StoneBC/bikes.json`
- Posts: edit `StoneBC/posts.json` directly
- Events: edit `StoneBC/events.json` directly
- Routes: GPX → `python3 Scripts/process_routes.py` → `routes.json`

## Bike Wait List Rule

When touching `inventory/` or flipping any bike to `status: "ready"`:
1. Check `inventory/waitlist.json` and `inventory/WAITLIST.md`
2. Run `python3 Scripts/match_waitlist.py` from StoneBC root — drafts notification emails
3. Never skip the matcher step

## Notes

- `Radio/` requires real devices for peer testing (MultipeerConnectivity)
- Rally Radio is a NavigationLink on Home tab, NOT a 5th tab (iOS 26 Liquid Glass hides 5th tabs)
- Expedition Journal stores media in iCloud Drive `8o7/` folder
- Trailforks integration needs API key in `config.json` — not bundled
