# StoneBC — Product Vision

**One-liner:** The local expert's guide to cycling South Dakota.

---

## The Problem

South Dakota has world-class cycling — Needles Highway, the Mickelson Trail, Black Hills gravel, Badlands panoramas — but no app that knows the terrain like a local. Strava tracks your ride the same way in Manhattan and Custer. It doesn't know which trails are closed, where cell service dies, which canyons swallow your signal, or which brewery is 2 miles off-route.

Rural cycling is fundamentally different from urban cycling. You can't call for help from a dead zone. You can't check trail conditions mid-ride without service. You can't coordinate your group through a canyon with no signal. General-purpose fitness apps weren't built for this.

## The Solution

StoneBC is a cycling companion built by people who ride these roads. Every route is curated, not algorithm-generated. Every dead zone is mapped. Every trail closure is pulled from USFS in real time. When your group splits at a fork, Rally Radio keeps you connected peer-to-peer — no cell tower needed.

**Core value:** Strava tracks your ride. StoneBC keeps you alive on it.

## Who It's For

**Primary:** Anyone cycling in South Dakota — tourists planning a trip, locals exploring new routes, bikepackers running multi-day tours, commuters navigating year-round.

**Secondary:** Stone Bicycle Coalition members — browsing refurbished bikes, volunteering, connecting with the co-op community.

## What Makes It Different

| Capability | StoneBC | Strava / Komoot / AllTrails |
|-----------|---------|---------------------------|
| Route curation | Hand-picked by locals who've ridden every mile | Crowdsourced or algorithm-generated |
| Offline capability | All routes bundled — works without signal | Requires connectivity for core features |
| Cell coverage mapping | Dead zones mapped so you know before you go | Not available |
| Group communication | Rally Radio — P2P voice chat, no cell needed | Not available |
| Trail conditions | USFS closures + crowdsourced reports | Not available |
| Emergency safety | Satellite SOS detection, 911 quick-access | Not available |
| Multi-day guides | Curated tour guides with stops, checklists, elevation | Route-only, no guide context |
| Ride documentation | Expedition Journal — photos, audio, GPS, narratives | Basic activity logging |
| Community bikes | The Quarry — refurbished bikes for the community | Not available |

## The Expansion Path

```
Phase 1: Own South Dakota — statewide routes, local partnerships, tourism board alignment
Phase 2: Adjacent states (WY, MT, NE, ND, CO) via regional cycling org partnerships  
Phase 3: Platform — any state or region deploys their own StoneBC instance
```

The model scales because the value is hyperlocal. Every state has roads Strava doesn't understand. Every rural region needs offline-first, safety-aware cycling tools. StoneBC proves the model in SD, then replicates it.

## Revenue Model

| Stream | Description |
|--------|-------------|
| Freemium subscription | Free: routes + basic recording. Premium: navigation, Rally Radio, offline maps, tour guides |
| Tourism partnerships | SD Tourism Board, Black Hills Visitors Bureau, regional CVBs |
| Sponsored routes | Local businesses featured as stops on curated routes |
| Event fees | Guided rides, bikepacking events |
| The Quarry | Community bike sales |
| State licensing | Other states/regions license the platform (Phase 2+) |

## Design Principles

1. **Offline-first** — Every essential feature works without signal
2. **Safety over social** — Cell coverage and SOS before kudos and leaderboards
3. **Curated over crowdsourced** — Quality routes vetted by locals, not quantity from algorithms
4. **Community-owned** — Co-op model, not VC extraction. LLC → 501(c)(3) → member cooperative
5. **Config-driven** — Any bike organization can fork, customize, and deploy
